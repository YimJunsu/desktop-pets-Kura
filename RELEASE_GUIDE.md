# 검은 고양이 데스크탑 펫 (Kuro Pet) 배포 및 설치 가이드 🐾

본 문서는 베타 테스터 배포 및 깃허브 Releases 등록 시 발생할 수 있는 주요 예외 사항과 이에 대한 해결 방안을 정리한 가이드라인입니다. 필요 시 테스터분들께 공지사항으로 그대로 공유하여 사용하실 수 있습니다.

---

## 🛡️ 1. Windows Defender SmartScreen 경고 대처법

설치 파일(`.exe`)을 실행할 때 아래와 같이 파란색 **"Windows의 PC 보호"** 창이 뜨는 것은 윈도우 보안 시스템의 표준 사양입니다.

### 💡 발생 원인
- Microsoft Defender SmartScreen은 전 세계에서 **다운로드 수(Reputation)가 적거나**, 수십만 원 상당의 **공인 인증서(EV Code Signing Certificate) 서명이 되어 있지 않은 실행 파일**을 실행할 때 PC 보안을 위해 이 경고를 띄웁니다.
- 개인 개발 또는 오픈소스 프로젝트의 경우 서명 비용의 한계가 있어 일반적으로 발생하며, 프로그램에 악성코드가 있다는 뜻이 아닙니다.

### ✅ 해결 방법 (베타 테스터용 안내)
1. 파란색 경고창 본문에 적힌 **[추가 정보]** 링크를 클릭합니다.
2. 아래에 나타나는 **[실행]** 단추를 누르면 정상적으로 설치 마법사가 실행됩니다.
3. *팁: 인스톨러 방식이 번거롭다면, 패키징 빌드 시 포터블(Portable) 무설치 ZIP 형태의 바이너리를 함께 압축해서 배포하시는 것도 SmartScreen의 차단을 피할 수 있는 훌륭한 대안입니다.*

## 🔄 2. 업데이트 탭의 "404 업데이트 서버 정보 확인 대기 중" 해결법

설정 창의 `UPDATER` 탭에서 업데이트 확인 시 **404 에러**가 발생하는 것은 깃허브 상에 다운로드할 파일 정보가 생성되지 않았기 때문입니다.

### 💡 발생 원인
- `electron-updater`가 작동할 때 `package.json`의 `publish` 경로에 적힌 깃허브 저장소를 스캔합니다.
- 하지만 현재 깃허브 저장소의 **Releases 탭에 퍼블리시(정식 릴리즈)된 버전이 단 하나도 없을 경우**, 관련 메타데이터 파일(`latest.yml`)을 찾지 못해 404 Not Found 에러가 발생합니다.

### ✅ 해결 방법 (개발자용 조치)
1. 깃허브 저장소(`YimJunsu/desktop-pets-Kura`)의 우측 **Releases** 영역으로 이동합니다.
2. **Draft a new release**를 누릅니다.
3. 태그 이름에 `v1.0.0` (반드시 `package.json`의 `version` 필드와 일치해야 함)을 적고 타이틀과 설명을 기입합니다.
4. `npm run dist` 결과물로 나온 `dist/` 폴더 내의 설치 파일들(예: `Kuro Desktop Pet Setup 1.0.0.exe`, `latest.yml` 등)을 첨부 파일 영역에 **모두 업로드**합니다.
5. **Publish release** 버튼을 눌러 정식으로 배포 상태로 전환합니다.
6. *참고: 비공개(Private) 저장소인 경우 업데이터 요청 헤더에 깃허브 토큰 인증이 필요하므로, 베타 배포용 저장소는 가능한 한 공개(Public)로 설정하여 배포하시는 것을 강력 권장합니다.*

## ⚙️ 3. 설정 창의 닫기 버튼 클릭 장애 관련 조치 완료
- 이전 버전에서 창을 드래그하기 위해 설정 창 전체 영역에 드래그 속성이 부여되어 버튼들이 클릭을 가로채 당했던 현상이 있었습니다.
- 현재 코드 개선을 통해 **설정 창 상단의 타이틀바 영역만 마우스로 드래그**할 수 있게 격리되었으며, 닫기(`X`) 버튼 및 하단의 `Done (닫기)` 버튼을 포함한 모든 입력창과 슬라이더 탭이 정상적으로 즉각 클릭 및 조작되도록 완전 복구되었습니다.
